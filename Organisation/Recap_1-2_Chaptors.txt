Computer Organization & Architecture – What We’ve Learned So Far
1. Basic Concepts
                Computer Organization → How things work internally (control signals, data paths, memory hierarchy, instruction cycles).
                Computer Architecture → What is visible to the programmer (instruction set, addressing modes, system design).

👉 Think of Organization = Implementation, Architecture = Design.

2. CPU & Memory Hierarchy
                          Registers → Fastest storage, inside CPU.
                          Cache (L1, L2, L3) → Stores frequently accessed data.
                          Main Memory (RAM) → Stores active programs/data.
                          Secondary Storage (HDD/SSD) → Permanent storage.
                          Principle of Locality → Temporal & Spatial locality → why caches work.
3. Performance Factors

                          Moore’s Law → More transistors in less space → faster chips.
                          Problems → Power consumption, heat, clock speed limits.
                          Solution → Multi-core processors, pipelining, parallelism.
4. Instruction Execution

          Instruction Cycle → Fetch → Decode → Execute → Store.
          Pipelining → Break instructions into stages, overlap execution.

Hazards
          Data hazards
          Control hazards (branching)
          Structural hazards

5. I/O & System Balance
          Processor power is very high, but I/O devices are slower.
Solutions:
                Buffers & Caches in memory.
                High-speed buses (PCIe, USB, etc.).
                DMA (Direct Memory Access) to bypass CPU for data transfer.

6. Memory Organization

                DRAM vs SRAM → DRAM slower but higher capacity, SRAM faster but expensive.
                Cache Organization → Direct-mapped, Fully associative, Set associative.
                Virtual Memory → Uses disk + RAM to give illusion of bigger memory.
